## v2.0.0 (2026-06-07) — 状态机引擎重构

> 🎯 目标：实现全自动循环出击，智能撤离，赢取更多星屑

### ⚡ 核心重写

- **状态机引擎**：`content.js` 完全重写为六状态自动化引擎
  - `MAIN_PAGE` → `START_CONFIRM` → `BATTLE` → `VICTORY_CHOICE` / `DEFEAT` → `WAREHOUSE` → 循环
  - 替代旧版简单的按钮文本扫描逻辑
- **真实 DOM 映射**：基于 `agentank.ai/raid` 真实页面 HTML 结构
  - 使用精确元素 ID（`raidStartBtn`, `raidRewardModal`, `raidEscapeBtn` 等）
  - 通过 `.raid-shell` 的 class（`is-run`, `is-warehouse`）判断页面阶段
  - 不再依赖模糊的 `textContent` 匹配

### 🧠 智能决策系统

- **撤离策略**（基于 remark.md 设计）：
  - 层 ≤ 3：总是继续挑战
  - 层 3 完成 + 无备用核心 → 撤离
  - 层 3 完成 + 备用核心 ≥ 1 → 继续到第 5 层
  - 层 5 完成 + 备用核心 < 2 → 撤离
  - 层 5 完成 + 备用核心 ≥ 2 → 继续到第 7 层
  - 层 ≥ 7 完成 → 总是撤离
- **强化技能选择优先级**：
  - 绝对优先：备用核心（只要出现就选）
  - 首次优先：未选过的技能各选一次
  - 重要度排序：自动护盾 > 宝物磁场 > 技能冷却 > 开局推进
- **仓库自动流程**：
  - 自动出售所有宝物（点击"全部出售"）
  - 星星为 0 时自动兑换 1 颗星星（只兑换最少量，保证能开始游戏）
  - 兑换后自动返回出击

### 🎨 Popup UI 升级

- **实时状态面板**：显示当前阶段、层数、星星、星屑余额
- **当前强化显示**：芯片样式展示已获得的强化技能及次数
- **战绩统计**：总出击数、胜利层数、撤离次数、失败次数、点击次数、运行时长
- **操作日志**：最近 20 条日志，按等级着色（info/warn/error/action/state）
- 移除旧版的"自动化配置"开关组（自动强化、备用核心等独立开关），改为全自动智能决策

### 🔧 技术改进

- **自适应轮询**：菜单 1.5s / 战斗 4s / 未知 3s，附 ±300ms 随机抖动
- **点击冷却**：1.8s 冷却防止重复点击
- **完整 Mouse Event 序列**：mouseover → mousedown → mouseup → click
- **状态同步**：通过 `chrome.storage.local` 实时同步游戏状态到 Popup

## v1.0.0 (2026-06-07) — 初始版本

### 功能

- Chrome Extension Manifest V3 基础框架
- 简单的按钮文本匹配 + 自动点击逻辑
- 通过 `findVisibleElementByText()` 查找页面按钮
- Popup 提供独立开关：自动强化、备用核心、开局推进、自动护盾、全部出售、兑换星星
- 基础统计：点击次数、运行时长
- 随机延迟点击（1-2.2s）模拟人类操作

### 已知问题

- 无状态感知，不知道当前处于哪个游戏阶段
- 无层级追踪，无法实现智能撤离
- 按钮匹配基于 `textContent` 容易误匹配嵌套元素
- 无仓库完整流程（出售 → 兑换 → 返回循环）
- 强化选择无优先级策略
