# 🤖 AgenTank Raid Helper (坦克出击 Raid 挂机助手)

[![Version](https://img.shields.io/badge/version-2.2.3-blue.svg)](https://github.com/xiedaibin/agentank-raid-helper)
[![Platform](https://img.shields.io/badge/platform-Chrome%20Extension%20v3-green.svg)](https://chrome.google.com/)

**AgenTank Raid Helper** 是一款专门针对 `https://agentank.ai/raid` 出击模式设计的智能全自动挂机 Chrome 浏览器插件（基于 Manifest V3 标准开发）。

本插件旨在通过高内聚的六状态机引擎与智能决策算法，实现全自动出击、智能坦克与技能选择、战术撤离、自动仓库出售宝物和星屑按需兑换星星的无限循环挂机。

> 🎯 **终极目标**：智能低风险锁分挂机上王者，长期胜率保持在 55% 以上。

---

## ✨ 核心特性

*   🤖 **全自动六状态状态机**：无需人工干预，自适应页面加载和延迟，自动识别并流转于“主页面检测 ➔ 坦克及出击确认 ➔ 战斗监控 ➔ 强化选择 ➔ 失败结算 ➔ 仓库管理”等六大状态。
*   🛡️ **智能战术撤离策略**：基于当前胜场与所拥有的“备用核心（复活命数）”进行低风险收益最大化评估：
    *   **≤ 3 层**：无条件继续，累积初始战斗力。
    *   **3 层挑战后**：若 `备用核心 == 0`，则在第三层结算时**撤离并保存**，避免无命暴毙；若 `备用核心 >= 1`，则挑战到第 5 层。
    *   **5 层挑战后**：若 `备用核心 < 2`，在第五层结算时**撤离并保存**；若 `备用核心 >= 2`，则挑战到第 7 层。
    *   **≥ 7 层**：挑战完成后**无条件撤离并保存**，锁定最大利益（保守锁分策略）。
*   🔮 **强化技能智能选择**：
    *   **绝对最高优先级**：`备用核心`（只要出现且未满 Lv.2，必选）。
    *   **首次拥有优先**：其他核心技能未获得过的（Lv.0），优先各获得一次，集齐 `自动护盾`、`宝物磁场`、`技能冷却`、`开局推进`。
    *   **二次升级排序**：`自动护盾` > `宝物磁场` > `技能冷却` > `开局推进`。
*   💰 **仓库资产自动流转**：
    *   **一键全部出售**：检测并自动依次点击“全部出售”或“出售”按钮，秒清宝物，回笼资金。
    *   **星屑智能兑换**：当 `星星 == 0` 时，自动在输入框填入 `"1"` 兑换一颗星星。按需小额兑换，防止一次性将星屑全部兑换成星星以降低星屑持有风险。
*   📊 **玻璃拟态侧边栏仪表盘**：
    *   实时显示挂机阶段、当前层数、星星与星屑余额。
    *   **星屑盈亏实时统计**：实时显示相对于本次挂机初始星屑的盈亏状态（绿色为盈，红色为亏，灰色为平）。
    *   **运行统计**：统计运行时长、累计点击数、出击次数、历史最高层数以及撤离与失败次数。
*   ⚙️ **控制台调试开关**：在代码头部可一键开启/关闭常规控制台调试日志输出，减少控制台资源占用。

## 🚀 安装与使用说明

### 1. 本地加载插件
1. 下载或克隆本项目到本地。
2. 打开 Chrome 浏览器，访问 `chrome://extensions/` 进入扩展程序管理页面。
3. 开启右上角的 **“开发者模式”** 开关。
4. 点击左上角的 **“加载已解压的扩展程序”** 按钮，选择本项目所在的根目录。
5. 看到具有赛博坦克图标的 **Agentank Raid Helper** 插件即表示加载成功。

### 2. 挂机运行
1. 打开挂机目标网页：[https://agentank.ai/raid](https://agentank.ai/raid)。
2. 页面右侧会自动展开玻璃拟态的控制面板侧边栏。
3. 确保侧边栏上的 **“主开关”** 处于开启状态，插件将自动检测当前页面状态并开始全自动挂机。
4. 如果需要暂停，随时在侧边栏或点击插件右上角图标关闭主开关即可。

### 3. 常规日志屏蔽与开启
*   如果您希望在控制台看自动化执行的日志：
    打开 `content.js`，将顶部的 `const ENABLE_LOG = false;` 修改为 `true`。
*   如果您希望保持控制台清爽：
    保持 `const ENABLE_LOG = false;` 即可。

## ⚠️ 避坑指南

1.  **上下文失效问题 (Context Invalidated)**：当您更新或重新加载插件时，旧页面中的定时器可能会失效并抛出 `Extension Context Invalidated` 异常。本插件在所有 API 调用前均加入了 `isContextValid()` 机制以实现无感销毁，杜绝崩溃。
2.  **点击穿透与事件触发**：游戏页面使用了虚拟 DOM，为防范假死，插件点击触发机制采用了完整的 `mouseover ➔ mousedown ➔ mouseup ➔ click` 事件序列来欺骗虚拟 DOM 树，并添加了点击冷却，防止快速连点报错。
